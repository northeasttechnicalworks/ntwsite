# docs/content-ops.md

Content lifecycle
- Draft -> Review -> Approve -> Publish
- Review includes: technical accuracy + brand tone + SEO

Update cadence
- Quarterly: services taxonomy review
- Monthly: proof artifacts refresh
- As needed: coverage updates

Governance rules
- No new service page without:
  - deliverables
  - example scope
  - related services links
  - FAQ block
  - diagram

Versioning
- Keep changes logged in a CHANGELOG.md (optional)

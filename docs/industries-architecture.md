# docs/industries-architecture.md

Industries must be described by:
- Typical deployments
- Operational constraints
- Decision criteria
- Proof artifacts that matter

Industries

Retail
Deployments: multi-site rollouts, cameras, access, network racks, cabling.
Constraints: overnight work, tight openings, standardized checklists.
Decision criteria: schedule reliability, consistency, closeout.
Required proof: rollout closeout sample; site checklist; before/after.

Commercial Office
Deployments: IDF buildouts, Cat6A, access control, cameras.
Constraints: tenant safety, clean work, low disruption.
Decision criteria: workmanship, documentation, professionalism.
Proof: labeling standard, rack photo standard.

Property Management / Multifamily
Deployments: intercom, access, cameras, MDF/IDF cleanup.
Constraints: tenant comms, phased installs.
Decision criteria: service discipline, clean installs.
Proof: service workflow and response.

Industrial / Warehouse
Deployments: camera coverage, long runs, fiber, rack stabilization.
Constraints: safety, lift work, harsh environments.
Decision criteria: safety compliance, durable installs.
Proof: industrial install photos, cable pathway methods.

Healthcare (if applicable)
Deployments: access, cameras, network infrastructure.
Constraints: strict windows, privacy, uptime.
Decision criteria: professionalism, zero disruption.
Proof: process discipline modules.

Education / Campus (if applicable)
Deployments: access, surveillance, paging support.
Constraints: school schedules, safety.
Proof: process + response.

Government / Municipal (if applicable)
Deployments: secure access, cameras, infrastructure upgrades.
Constraints: procurement, compliance.
Proof: documentation + security posture.

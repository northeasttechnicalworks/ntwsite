# docs/seo-strategy.md

SEO objectives
- Rank for high-intent service + location terms in NY/NJ/CT
- Rank for integrator coverage terms
- Build authority via documentation/testing/remediation content

Keyword mapping rules
- One primary keyword cluster per page
- Each page has: H1 = primary intent, H2s = supporting intent

Schema strategy
- Home: Organization JSON-LD + Website JSON-LD
- Service pages: Service JSON-LD + BreadcrumbList
- FAQ sections: FAQPage JSON-LD
- Coverage: LocalBusiness (optional) + areaServed + BreadcrumbList

Internal linking
- Service pages link to:
  - related services (3)
  - industries (2)
  - coverage
  - contact
- Industries pages link to:
  - top 4 relevant services
  - integrators (if relevant)
- Integrators links to:
  - coverage
  - services
  - proof artifacts
  - contact

Content clusters (recommended)
- Structured cabling: labeling, certification, MDF/IDF best practices
- Remediation: rack cleanup, failed installs rescue
- After-hours cutovers: planning and checklist
- Rollouts: multi-site execution framework
- Documentation: closeout packages, test results

Meta templates
(Use templates from report section)

Page indexing rules
- No thin pages (< 400 words). If short, add FAQ + proof blocks.

Slug conventions
- /services/{service}
- /services/{service}-{state}
- /industries/{industry} (optional if expanding)
- /resources/{topic} (optional later)
